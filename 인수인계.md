## 1. 프로젝트 개요

PUBG 유튜브 썸네일 제작을 위한 Electron 데스크탑 앱.
두 가지 핵심 기능으로 구성됨:

1. **썸네일 추출기** — 영상에서 AI로 최적 컷 자동 추천
2. **썸네일 편집기** — 추출된 프레임에 PUBG 브랜드 템플릿/타이포 합성 후 PNG 저장

---

## 2. 실행 방법

```bash
# 앱 폴더에서
npm start

# 또는 더블클릭
썸네일워크벤치_실행.command
```

> **Node.js / npm 필요** — 없으면 https://nodejs.org 에서 설치
> 처음 실행 시 `npm install` 먼저 실행

## 6. 썸네일 추출기 스펙

### 스캔 모드

| 모드 | 추출 | 분석 | 방식 |
|---|---|---|---|
| 빠른 추천 | 최대 72컷 | 상위 24컷 | 픽셀 프리필터 → Gemini |
| 고급 전수 | 최대 120컷 | 상위 30컷 | 픽셀 프리필터 → Gemini |
| Video-AI | 영상 전체 | AI 추천 구간 | Gemini File API 직접 분석 |

- 샘플링 범위: 영상 길이 **10%~90%** (인트로/아웃트로 제외)
- 병렬 처리: Gemini API 최대 **8개 동시** 요청

### 픽셀 프리필터 점수 (모델 없음, 순수 픽셀 계산)

| 지표 | 가중치 | 비고 |
|---|---|---|
| 밝기 | 20점 | ITU-R BT.709 가중치, 목표 0.6 |
| 대비 | 20점 | 밝기 분산의 제곱근 |
| 채도 | 15점 | HSV (max-min)/max |
| 엣지 밀도 | 10점 | 인접 픽셀 차이 80 이상 |
| 액션 델타 | 27점 | 전후 프레임 중앙 50% RGB 차이 |
| 웜톤 비율 | 8점 | R/G > 160 && > B×1.4 |
| sharpness | 제거됨 | 모션블러 액션 컷 보호 목적으로 의도적 제외 |

> ⚠️ **캐릭터 유무는 픽셀 프리필터로 판별 불가** — 캐릭터 관련 피처는 전부 Gemini 단계에서만 추출됨
> 향후 개선 검토 사항: 경량 로컬 모델(YOLO/MediaPipe) 추가 또는 저해상도 Gemini 1차 필터 도입

### Gemini 채점 공식

```
최종점수 = Base(0~100) + Rules보너스 + 콘텐츠유형보너스(-50~+55)
```

**Base 점수 카테고리:**
- A. 씬 유형 (0~50): combat+14 / victory+10 / lobby-16
- B. 캐릭터 (-10~20): large+10 / dynamic+6 / single+8 / crowded-8
- C. 구도 (-5~15): 삼분할+5 / deep+6 / 여백+4
- D. 액션 (0~10): high+10 / medium+5
- 패널티 (최대-30): UI high-25 / 비게임플레이-14

**콘텐츠 유형 보너스:**

| 유형 | 핵심 기준 |
|---|---|
| gameplay | 단독주인공+28 / 교전+28 / 액션high+18 |
| cinematic | 비게임플레이+22 / UI없음+18 / 삼분할+18 |
| teasing | silhouette+50 / partial+30 / full-15 |
| new_item | prominent+55 / none-28 |
| celebrity | 실존인물+45 / 얼굴+28 |
| artist_collab | large_group+55 / solo-22 |

### 후처리

- **중복 제거**: 최소 간격 3.0s → 2.0s → 1.5s → 1.0s → 0.5s 자동 완화 (목표 12컷)
- **대기 풀**: 프리필터 탈락 프레임 보관 → "더 보기"로 추가 재분석 가능
- **주변 탐색**: 선택 컷 ±0.5초 0.1초 간격 11컷 미리보기
- **Fine sampling**: 타겟 ±2.0초를 0.2초 간격 탐색, 가장 근접 컷 반환

## 7. 썸네일 편집기 스펙

### 캔버스

- **해상도**: 1920 × 1080px
- **내보내기**: PNG

### 레이어 합성 순서 (z축)

```
1. 검정 배경
2. 베이스 프레임 이미지  (커버 스케일, 위치/크기/회전 조정 가능)
3. 딤 오버레이           (하단 그라디언트, 투명도 조절)
4. 템플릿 PNG            (Basic 또는 Trailer)
5. 타이포그래피
```

### 이미지 조정 컨트롤

| 컨트롤 | 범위 | 기본값 |
|---|---|---|
| 크기 | 0.5× ~ 2.0× | 1.0× |
| X 위치 | -960 ~ +960px | 0 |
| Y 위치 | -540 ~ +540px | 0 |
| 회전 | -180° ~ +180° (0.5° 단위) | 0° |

### 딤 효과

- 캔버스 상단 35% 지점 ~ 하단 검정 선형 그라디언트
- 불투명도 0~100% 슬라이더
- 기본값 0% (비활성)

## 10. 향후 개선 검토 사항

1. **캐릭터 감지 프리필터** — 현재 픽셀 프리필터는 캐릭터 유무 판별 불가.
   후보 옵션:
   - 경량 ONNX/WASM 모델 (YOLOv8n 등) 로컬 실행
   - 저해상도 이미지로 Gemini 1차 간이 필터 ("캐릭터 있음/없음만 판별")
   → **단기 추천: 저해상도 Gemini 1차 필터** (구현 빠르고 정확도 보장)

2. **Basic PNG 투명도** — `YT_Thumbnail_Basic_16x9_EN.png`가 흰 배경으로 저장되어 있음.
   디자인 툴에서 투명 배경으로 재익스포트하면 현재 source-over 합성이 정상 동작함.

3. **PUBGRegular 폰트** — 로드만 되고 현재 미사용. 향후 타이포 확장 시 활용 가능.

4. **Rules 피드백 루프** — 고성과 썸네일 데이터가 쌓이면 채점 가중치 자동 보정 구조 이미 있음. 데이터 누적 후 활용 검토.
